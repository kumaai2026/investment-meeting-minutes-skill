# 投资会议纪要 - 上市公司交流

先读取并遵守同一 skill 包内的基础 skill：`投资会议纪要整理`；如果导入环境使用英文别名，则读取 `investment-meeting-minutes`。不要依赖 macOS 绝对路径。

跨系统导入要求：本文件必须保持 UTF-8 无 BOM、LF 换行，并通过基础 skill 的 `validate_utf8_text.py --portable-skill` 校验；Windows 导入时不得保存为 ANSI/GBK。

本类型的额外规则：

- 元信息必须保留 `**会议标题**` 和 `**会议类型**：上市公司交流`。
- 如果本次整理提供了 `segmentation_plan`、`target_attribution_ledger`、`evidence_ledger` 或 `suspect_confirmation` sidecar，只把它们作为内部依据；不得把 sidecar 字段、JSON、角色名或处理过程写入终稿正文。
- 首要原则是最大限度保留管理层、主持人和投资者原文，只剔除无意义语气词、无意义重复和明显 ASR 噪声；不得把长回答改写成短摘要。
- 如果会议只讨论一家公司，顶部标题必须承载板块、标的和代码，格式优先使用：`# 投资会议纪要｜板块｜标的(代码)`。如果板块、标的或代码不能确认，用原文表述或 `待确认`，并写入存疑。
- 元信息中增加 `**会议标的**：板块｜标的(代码)`；不得只在发言人小段里出现标的信息。
- 本类型覆盖基础 skill 中“每个原文小段标题都使用 `【板块｜标的(代码)】`”的要求：公司级板块/标的/代码已经放在标题和元信息后，各发言人部分不再重复展示这些信息。
- `## 一、逐发言人原文整理` 可以按公司交流自然结构组织：`公司概况/经营情况/业绩与指引/订单与客户/产能与产品/问答/风险与待确认`，但仍要保留发言人或问答来源。
- 发言人下的小段标题只写业务主题或问题主题，例如 `【经营】`、`【订单】`、`【产能】`、`【毛利率】`、`【客户】`、`【风险】`，不要重复 `板块｜标的(代码)`。
- 即使正文小段标题不重复公司级标的，也必须在内容中区分会议标的、客户、供应商、竞争对手、对标公司和行业背景标的；不得把客户、供应商、竞争对手或对标公司误写成推荐对象。
- 如果问答中出现对另一个标的的明确推荐、看多、看空、重点关注、买入、卖出、加仓或减仓，必须在对应问答正文中体现该真实推荐标的；必要时把该问题拆成独立小段。
- 如果公司级标的、问题主语和推荐对象不一致，优先保留原文关系，并将无法确认的标的归因写入存疑。
- 管理层、调研对象、提问者、回答者的口径要区分清楚；不把回答加工成自己的结论。
- 如果公司交流中出现问答，问答部分使用专家交流模板：问题加粗，回答以 `【】` 主题标签开头，标签后直接接回答正文，不写 `A：` 或 `A:`；标签不限定字数但必须概括回答主旨方向；不得使用 `问答 1` 这类序号标题。
- 优先保留收入、利润、毛利率、订单、产能、价格、客户、产品进度、资本开支、业绩指引和时间节点。
- 对没有明确来源的代码、客户名、产品名、数字放入 `## 二、存疑与待确认`。
- 对所有存疑内容同时结合音频/转录、上下文、`a-stock-data` 标的证据和专业名词搜索核验；客户名、产品型号、技术路线、产能单位、政策/事件词、机构/人物名称不能只靠 ASR 猜测。若搜索证据、公司语境和音频听感无法统一，保留原文表述并写入存疑表。
- 整理后的每条真实存疑都必须完成上下文 + 搜索/证据核验，并在 `核验依据` 列写明 `上下文：...；检索/证据：...`，不能只写“待确认”或只依赖 ASR。
- 对公司名、代码、客户名、产品名、数字、订单、产能、价格、毛利率、收入、利润和标的角色归因，不能确认时必须进入存疑表，并给出上下文和证据路径。
- 所有存疑片段必须在原文中同步标注：Markdown 中用 `**存疑词**` 加粗；只有当前会话含音频/视频或带时间戳转写时，才追加 `（存疑时间戳：HH:MM:SS）`。文稿/文档-only 会话正文不写时间戳占位，但存疑表 `时间戳` 单元格写 `未提供`。导出 Word 时必须显示为加粗 + 下划线；这是所有会议类型都要遵守的基础规则。
- `## 二、存疑与待确认` 只在存在真实存疑内容时输出；表头固定为 `时间戳 | 原始表述 | 当前判断 | 存疑原因 | 候选项 | 核验依据 | 人工确认`。
- 不输出结构化摘要和标的表格；这些只在人工确认终稿后由后续脚本生成。

输出骨架：

```markdown
# 投资会议纪要｜板块｜标的(代码)

**会议日期**：YYYY-MM-DD
**整理时间**：YYYY-MM-DD
**会议标题**：原始会议标题
**会议类型**：上市公司交流
**会议系列**：研究所周会
**会议标的**：板块｜标的(代码)

---
```
