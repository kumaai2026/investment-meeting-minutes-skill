# 投资会议纪要 - 专家交流

先读取并遵守同一 skill 包内的基础 skill：`投资会议纪要整理`；如果导入环境使用英文别名，则读取 `investment-meeting-minutes`。不要依赖 macOS 绝对路径。

跨系统导入要求：本文件必须保持 UTF-8 无 BOM、LF 换行，并通过基础 skill 的 `validate_utf8_text.py --portable-skill` 校验；Windows 导入时不得保存为 ANSI/GBK。

本类型的额外规则：

- 元信息必须保留 `**会议标题**` 和 `**会议类型**：专家交流`。
- `## 一、逐发言人原文整理` 必须采用问答形式。优先保留原文中的 `q:` / `a:`、提问者/回答者、主持人/专家/追问关系；如果原文没有明确 Q/A，也要按语义整理成连续问答，但不得编造来源。
- 首要原则是最大限度保留专家和提问者原文，只剔除无意义语气词、无意义重复和明显 ASR 噪声；不得把长回答压缩成短摘要。
- 问题部分必须加粗，格式使用 `**Q：问题原文**` 或 `**提问：问题原文**`。
- 回答部分必须以一个 `【】` 包裹的主题标签开头，后面直接接回答正文，例如 `【需求变化】...`、`【价格压力】...`、`【产能瓶颈】...`、`【验证风险】...`。标签不限定字数，但必须能概括回答主旨方向，不能写成空泛词。
- 回答正文不得出现 `A：` 或 `A:` 前缀；`【主题】` 后直接输出内容即可。
- 不使用 `### 问答 1`、`### 问答 2` 这类序号标题；通过空行、加粗问题、主题标签和必要的发言人标签区分不同问答即可。
- 每个回答都保留专家判断依据：数据来源、产业链位置、客户反馈、渠道口径、产能/订单/价格/利润等。
- 专家观点必须带判断依据：数据来源、产业链位置、客户反馈、渠道口径、产能/订单/价格/利润等。
- 明确区分事实、专家判断、推测和待验证事项。
- 对专家口径中的行业术语、公司简称、客户名称、数字和比例进行校准；不稳妥则标存疑。
- 所有存疑片段必须在原文中同步标注：Markdown 中用 `**存疑词**` 加粗，并紧贴标注对应录音时间段，例如 `**存疑词**（录音：00:12:34-00:13:08）`；无可用时间戳时写 `（录音：无可用时间戳）`。导出 Word 时，存疑词必须显示为加粗 + 下划线；这是所有会议类型都要遵守的基础规则。
- 不输出结构化摘要和标的表格；这些只在人工确认终稿后由后续脚本生成。

输出骨架：

```markdown
# 投资会议纪要

**会议日期**：YYYY-MM-DD
**整理时间**：YYYY-MM-DD
**会议标题**：原始会议标题
**会议类型**：专家交流

---

## 一、逐发言人原文整理

**Q：保留清理后的问题原文。**

【需求变化】保留清理后的专家回答原文，保留依据、数字、条件、边界和不确定表达。

**Q：追问原文。**

【验证风险】回答原文。

---

## 二、存疑与待确认

| 原始表述 | 当前判断 | 存疑原因 | 候选项 |
| --- | --- | --- | --- |
```
