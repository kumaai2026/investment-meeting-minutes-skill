# 证据规则

本文件定义信息核验的证据层级和边界。证据结论供 subagent 中间产物和主流程成稿使用，不得直接变成最终纪要中的流程字段。

## 证据层级

### source_evidence

来源：
- 原始音频
- SenseVoice 转写
- 带时间戳转写 JSON
- 用户上传文档
- 人工初审稿
- 同会话补充说明

地位：
- 是会议内容最高优先级证据。
- 只能据此整理会议中实际说过的判断、数字、动作和不确定表达。
- 外部资料不能替代 source evidence 编造会议结论。

### local_reference_evidence

来源：
- `query_symbol_candidates.py`
- 本地 A 股、港股、美股代码表
- 公司别名表
- 本地术语表或历史归档中可用的候选信息

地位：
- 只用于候选纠错和缩小检索范围。
- 本地候选不是最终确认。
- 只有本地候选、外部尚未确认时，不得标记 `confirmed`。
- 置信不足、候选冲突或资源过期时必须保留为存疑或继续外部核验。

### external_verification_path

来源：
- 公司公告
- 交易所披露
- CNINFO
- 公司官网
- 监管/协会资料
- 行业报告
- 可靠财经新闻
- `a-stock-data` live/network data

地位：
- 辅助确认实体、代码、术语、客户公开信息、订单/产能/价格口径。
- 不得用于添加会议没有出现的新投资结论。
- 与 source evidence 冲突时，保留原始表述并列为存疑。

## 高风险 claim

以下信息属于高风险事实，写入终稿前必须核验：
- 公司名和股票代码
- 客户名、供应商名、竞争对手名
- 订单、产能、价格、收入、利润、毛利率、估值、市值
- 时间节点、政策事件、公告事件
- 产品型号、技术路线、专业术语
- 投资动作：买入、卖出、加仓、减仓、重点跟踪

subagent 或主流程记录高风险事实时，至少保留：

```json
{
  "claim_id": "",
  "claim_type": "",
  "claim_text": "",
  "source_ref": "",
  "source_evidence": "",
  "external_verification_path": "",
  "evidence_status": "",
  "final_handling": ""
}
```

## evidence_status

- `confirmed`: source 与候选/外部证据一致。
- `candidate_only`: 只有本地候选或上下文候选，不能写成确定事实。
- `conflicting`: source、文稿或外部证据互相冲突。
- `unverified`: 尚未核验。
- `not_found`: 已检索但未找到可靠证据。
- `not_applicable`: 该 claim 不需要外部确认，例如发言人个人观点。

`candidate_only` / `unverified` / `not_found` 的 ticker 不得作为已确认代码写入最终标题。若正文必须保留候选代码，必须在正文和存疑表中明确标注。

## 存疑表要求

每条真实存疑必须写明：
- 上下文：为什么该候选符合或不符合当前段落。
- 检索/证据：查了什么、得到什么、为什么仍不能确认或如何确认。
- 人工确认：留空。

不得只写：
- `待确认`
- `需核实`
- `未查到`
- `听不清`

这些词可作为存疑原因的一部分，但不能替代上下文和证据路径。

## 稳定核验 Prompt

当出现非人名存疑项时，主流程或 subagent 必须使用以下 prompt 进行同一套核验。不要新增脚本或 validator 来证明核验发生；核验结果由最终存疑表和人工复核承接。

```text
你是中文投研会议纪要的存疑项核验助手。请只核验当前会议材料中已经出现的非人名存疑项，不补写会议没有说过的新结论。

输入：
1. 会议上下文：相关原文段落、发言人/版块、时间戳或来源位置。
2. 存疑原始表述：ASR 或文稿中的原始词句。
3. 候选解释：可能的公司名、股票代码、简称、英文缩写、客户/供应商、数字、日期、产品、政策或事件。

核验要求：
1. 先用会议上下文判断候选是否语义匹配。
2. 再使用至少一种可靠证据路径核验：联网搜索、a-stock-data、公司公告、官网、交易所/监管披露、研报/行业资料、本地代码表。
3. 本地候选和 ASR 输出只能作为线索，不能单独作为确认依据。
4. 外部资料只能用于确认名称、代码、术语、公开客户/供应商、事件或数字口径，不能补写会议没有出现的观点。
5. 如果证据冲突、证据不足或无法唯一确认，保留原始表述并进入存疑表。

输出每条存疑项：
- 原始表述：
- 当前判断：确认 / 保留原文 / 暂不采信 / 待人工确认
- 存疑原因：
- 候选项：
- 核验依据：上下文判断 + 实际使用的证据路径 + 核验结论
- 最终处理：写入正文 / 正文加粗标注并入存疑表 / 不写入正式标的
```

## 不可越界

- 不用外部资料补写原文没有的客户名、订单额、业绩预测。
- 不把公开资料中的确定事实伪装成会议发言。
- 不因为外部资料更完整就覆盖音频/文稿中的明确不确定表达。
- 不把传闻、小作文、未公开客户名写成确定事实。
