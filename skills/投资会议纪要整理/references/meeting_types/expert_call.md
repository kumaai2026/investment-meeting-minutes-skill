# 专家交流 Reference

本文件定义 `专家交流` 的最终纪要格式依据。共同元信息、存疑表和 Word 导出规则仍以 `../output_contract.md` 为准。

## 适用范围

- 专家访谈、专家电话会、产业链深访和围绕专家经验展开的问答交流。
- 讨论对象可以是公司、产业链、技术路线、订单、客户或渠道情况。

## 正文结构

- `## 一、发言整理` 必须采用问答结构。
- 问题和回答必须分开，问题中不得混入回答内容。
- 回答直接接在对应问题后，保留专家原始逻辑展开。
- 不额外包装为摘要小标题。

## 原文整理规则

- 保留专家判断依据、数据来源、产业链位置、客户反馈、渠道口径、产能、订单、价格和利润边界。
- 长回答不得压缩成短摘要，不得改写为研报段落。
- 不把回答改成“专家表示”“专家认为”等第三人称转述口吻。
- 专家口径中的不确定、估计、传闻、未披露客户和未核实型号必须保留边界或进入存疑。

## 格式示例

```markdown
## 一、发言整理

### 主持人/专家

**【设备订单现在是什么状态？】**

订单能见度是有改善的，但交付节奏还是要看客户扩产安排。这个判断主要来自近期产业链排产反馈和设备厂排期变化，不是公司正式指引。

**【哪些信息还不能确认？】**

具体订单金额、客户名称和单季度收入确认节奏都还需要继续确认，不能直接外推成全年利润预测。
```
