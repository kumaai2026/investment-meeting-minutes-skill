# Word 导出版式参考

参考样例：
`/Users/kumaai/Documents/Codex/workspace/投资纪要工作流/04 Archive/测试用/2026-04-27 - 投资会议纪要-舵主-标注版.docx`

本文件记录该样例已确认的输出风格。后续生成 `.docx` 时优先遵守本参考。

## 文档结构

Word 文档应使用正式报告式排版，不要把 Markdown/YAML 符号原样写入 Word。

标题与章节顺序：

1. 标题页/标题区：`投资会议纪要`
2. 元信息表：会议日期、整理时间、会议标题、会议类型、会议系列，以及类型 skill 要求的读者必要字段
3. `一、逐发言人原文整理`
4. `二、存疑与待确认`

最终 Word 纪要面向人阅读，不展示 `输入来源`、`整理说明`、上传文件名、工具日志等流程信息。

不要在章节标题后追加 `（人工格式）`、`（集中查看）` 等括号说明。

## 原文整理尺度

`一、逐发言人原文整理` 是“校对后的原文”，不是逐字转录，也不是 analyst summary。

必须做到：
- 严格按实际发言顺序排列。
- 最大限度保留原文，只删除口水词、无意义重复和明显 ASR 噪声。
- 断句应通顺，必要时可把过长口播拆成自然段。
- 保留原发言中有意义的判断、理由、数字、时间点、催化、业绩预期、仓位动作、条件句和不确定表达。
- 保留原发言人的视角、人称和语气，不把“我/我们/咱们”改写成第三人称旁白。
- 尽量保留发言人的原始措辞、语序、判断边界和口语化表达；不要把能读懂的原文改写成整理者自己的语言或研报式表述。
- 不得编造原文没有的信息。
- 不得把发言人的观点替换为自己的总结。

示例风格：

```markdown
【PCB/光模块链｜东山精密(002384.SZ)】
我刚才看了东山精密一季报，觉得挺好，应该落在预期上沿。比如在建工程增加了十个亿，这点比较好，说明扩产力度很猛。

公司刚刚合并索尔思光电，当前费用率比较高，还有汇率损失和计提等影响。我看过一个分析，如果这些因素回归，净利率差不多能到 30%。索尔思光电这块本质上还是看扩产，所以后面要看利润率变化。
```

## 存疑标注

所有进入 `二、存疑与待确认` 的原始表述，若在 `一、逐发言人原文整理` 中出现，必须同步标注。

Markdown 中：
- 用 `**存疑词**` 标记，并紧贴标注对应录音时间段，例如 `**存疑词**（录音：00:12:34-00:13:08）`。
- 只有段落级或字幕级时间戳时，使用可回听的最小可用范围；无音频或无可用时间戳时，写 `（录音：无可用时间戳）`，并在存疑表说明原因。

Word 中：
- 用“加粗 + 下划线”标记。
- 录音时间文字本身不需要加粗或下划线。
- 可使用深色或棕色强调，但不要只依赖颜色。

标注示例：
- `疑似“**西字科技**（录音：00:12:34-00:13:08）”`
- `**FAO/IAU**（录音：00:18:10-00:18:42）`
- `疑似“**结创智能**（录音：无可用时间戳）”`
- `**DLV**（录音：01:02:03-01:02:35）`

## Word 排版

整体风格：
- 正式、干净、适合扫描。
- 使用清晰标题层级；章节标题用深蓝色突出。
- 元信息使用两列表格，左列为字段名，右列为内容。
- 板块/标的标题使用浅色底纹条。
- 正文段落使用正常行距，避免密集堆叠。

表格：
- 元信息表和 `二、存疑与待确认` 使用真正 Word 表格。
- 表头使用深蓝底色、白字；正文行可使用浅色隔行底纹。
- 不使用 PDF 作为最终导出物；PDF/PNG 仅用于内部版式检查。

## 导出与质检

Word 导出后必须做版式检查：
- 首选 LibreOffice/`soffice` 渲染 DOCX。
- 使用 Poppler/pdf2image 或 `render_docx.py` 生成页面 PNG。
- 抽查首页、原文正文页、存疑表页。
- 检查是否存在：文字重叠、表格溢出、列宽挤压、页眉页脚异常、存疑标注缺失。

若本机缺少渲染依赖，应安装 LibreOffice 和必要的 PDF 渲染工具后再检查。
