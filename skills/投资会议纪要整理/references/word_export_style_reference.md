# Word 导出版式参考

本文只规定 Word/DOCX 版式。正文结构、标题、存疑表列和会议类型格式以 `output_format_guide.md` 为准。

## 结构

- Word 文档使用正式报告式排版，不把 Markdown/YAML 符号原样写入。
- 顺序：标题区 `投资会议纪要`、元信息表、`一、发言整理`、真实存疑存在时追加 `二、存疑与待确认`。
- 不展示 `输入来源`、`整理说明`、上传文件名、工具日志、路径等流程信息。
- 不在章节标题后追加 `（人工格式）`、`（集中查看）` 等说明。

## 样式

- 标题和章节标题使用清晰层级，章节标题用深蓝色突出。
- 元信息使用两列表格，左列为字段名，右列为内容。
- 板块标题和 `标的：` 行分行展示；板块标题可使用浅色底纹条，标的行使用较小字号或普通正文样式，避免挤成一行。
- 正文段落使用正常行距，避免密集堆叠。
- 元信息表和存疑表必须使用真正 Word 表格；表头使用深蓝底色、白字，正文行可使用浅色隔行底纹。

## 存疑标注

- Markdown 中的 `**存疑词**` 导出到 Word 后必须加粗并加下划线。
- 含音频/视频或带时间戳转写时，正文中对应存疑片段后必须保留同一存疑时间戳。
- 文稿/document-only 时，正文不写时间戳或 `未提供` 占位。

## 导出与质检

- 最终交付不生成 PDF；PDF/PNG 只用于内部版式检查。
- Word 导出后首选用 LibreOffice/`soffice` 渲染，再用 Poppler/pdf2image 或 `render_docx.py` 生成页面 PNG。
- 抽查首页、正文页和存疑表页，确认无文字重叠、表格溢出、列宽挤压、页眉页脚异常或存疑标注缺失。
- macOS 上若 LibreOffice 缺少 `little-cms2` / `liblcms2`，启动相关服务时设置 Homebrew 库路径到 `DYLD_FALLBACK_LIBRARY_PATH` 与 `DYLD_LIBRARY_PATH`。
